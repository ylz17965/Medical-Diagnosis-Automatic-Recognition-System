### 第3页：项目背景

**医疗健康咨询现状**

- 医疗资源分布不均，优质医疗资源集中在大城市
- 患者健康咨询需求旺盛，但专业医生时间有限
- 网络医疗信息良莠不齐，患者难以辨别真伪
- 传统AI医疗助手缺乏可解释性，信任度低

**市场数据**：
- 中国互联网医疗市场规模：2000亿元+
- 在线问诊用户规模：3亿+
- 医疗AI市场年增长率：40%+

### 第7页：知识图谱架构

**医学知识图谱 Schema**

| 实体类型 | 数量 | 说明 |
|---------|------|------|
| 疾病 | 5,000+ | ICD-10编码 |
| 症状 | 8,000+ | 标准化术语 |
| 药品 | 12,000+ | 国家药典 |
| 检查 | 3,000+ | 检验检查项目 |
| 科室 | 50+ | 临床科室 |

**关系类型**：12种
- HAS_SYMPTOM（疾病-症状）
- TREATED_BY（疾病-药品）
- NEEDS_EXAMINATION（疾病-检查）

### 第8页：RAG检索增强

**混合检索架构**

```
用户查询
    │

    │         ↓
    │     文档分块 → Embedding → 向量数据库
    │
    ├──→ 图谱检索（结构化知识）
    │         ↓
    │     实体识别 → 图谱查询 → 关联推理
    │
    └──→ 融合排序
              ↓
          重排序模型
              ↓
          最终结果
```
