# 智疗助手 — 演示视频脚本

## 基本信息

- **时长**: 3 分钟
- **演示页面**: 登录 → 聊天(深度搜索/报告/药盒) → CT 三维可视化 → 知识库管理 → 个人中心
- **分辨率**: 1920×1080
- **背景音乐**: 轻科技感 BGM（可选）

---

## 分镜脚本

### ① 开场 + 登录 (0:00 — 0:15)

**旁白**:

**操作**:
1. 输入手机号 `13800138000`，密码 `Test123456`
2. 点击"登录"

---

### ② 深度搜索问答 (0:15 — 0:55)

**画面**: 进入聊天主页面 → 点击"深度搜索"卡片

**旁白**:
> "登录后进入主界面，点击**深度搜索**进入智能问答模式。我们先问一个医学问题。"

**操作**:
1. 点击 FeatureEntry → **深度搜索**（search）卡片
2. 输入问题：`肺部磨玻璃结节8mm需要手术吗？`
3. 等待 AI 流式回答
4. 展示回答中的引用部分

**旁白**（AI 回答期间）：
> "系统调用了**阿里云百炼 qwen-max 模型**，结合 RAG 知识库检索结果生成带引用的专业回答。每个结论都有来源标注，点击可以查看详情。"

**画面**: 展示流式输出的文字逐字出现效果 + 底部引用区域

---

### ③ 切换模式 — 快速问答 (0:55 — 1:15)

**画面**: 点击模型选择器 → 切换到"快速回答"

**旁白**:
> "系统支持三种模型模式：智能选择、深度分析和快速回答。我们切换到**快速回答**模式，使用的是 qwen3.5-flash 模型，响应更快。"

**操作**:
1. 点击输入框上方的 **"智能选择"** 按钮
2. 选择 **"快速回答"**
3. 输入：`感冒发烧了应该怎么办？`
4. 观察回答速度明显更快

**旁白**:
> "Flash 模型适合日常健康咨询，响应速度是 Max 的 3-5 倍。"

---

### ④ 肺部 CT 三维可视化 (1:15 — 1:45)

**画面**: 导航到 `/lung-ct` → 点击"模拟肺部CT"

**旁白**:
> "接下来演示项目的核心技术——**肺部 CT 三维体渲染**。点击加载模拟数据。"

**操作**:
1. 点击侧边栏导航到肺部 CT 页面（或直接点击 LungCTView 按钮）
2. 点击 **"模拟肺部CT"** 按钮
3. 等待 3D 渲染完成

**旁白**（渲染过程中）：
> "系统基于 **VTK.js** 在浏览器端实现三维体渲染，无需安装任何插件。"

**画面**: 鼠标交互演示
- **左键拖拽**旋转视角
- **滚轮**缩放
- 切换窗宽窗位：点击"肺窗""纵隔窗""骨窗"

**旁白**:
> "我们可以用鼠标左键旋转模型、滚轮缩放，还可以切换不同的窗宽窗位——肺窗、纵隔窗、骨窗，从不同角度观察肺部组织结构。"

**操作**:
1. 左键旋转 + 滚轮缩放
2. 点击切换预设：**肺窗** → **纵隔窗** → **骨窗**

---

### ⑤ 知识库管理 (1:45 — 2:15)

**画面**: 导航到 `/knowledge` → 展示已有文档列表

**旁白**:
> "接下来是**知识库管理**系统。这里存储了 AI 问答所使用的医学知识文档。"

**操作**:
1. 展示文档列表（统计卡片：文档总数、分块数量、分类数）
2. 点击 **"添加文档"**
3. 填写：
   - 标题：`COVID-19诊疗方案（试行第十版）`
   - 来源：`国家卫健委`
   - 分类：`guideline`
   - 内容：`新型冠状病毒感染（COVID-19）的诊疗方案......`
4. 点击"添加"

**旁白**:
> "添加文档后，系统会自动对文档内容进行**智能分块**，生成向量索引。AI 回答时就能检索到这条资料了。"

**操作**:
1. 展示搜索功能：在搜索框输入 `结节`，展示搜索结果
2. 展示批量删除：勾选 2-3 个文档 → 点击"批量删除" → 确认

**旁白**:
> "知识库支持全文搜索和分类筛选，还支持批量删除等管理功能。"

---

### ⑥ 个人中心 + 总结 (2:15 — 2:40)

**画面**: 导航到 `/profile` → 展示个人信息设置

**旁白**:
> "最后看一下**个人中心**。这里可以管理基本资料、修改密码，以及配置 API 密钥。"

**操作**:
1. 展示个人信息页
2. 展示 API 密钥配置区域（Qwen API Key / 模型选择）

**旁白**:
> "用户可以在模型配置中自定义 API Key 和模型参数，支持 BYOM（自带模型）模式。"

---

### ⑦ 结尾 (2:40 — 3:00)

**画面**: 返回首页 → 展示系统架构

**旁白**:
> "总结一下：智疗助手的技术亮点包括——

> **第一**，基于 **pgvector + 阿里云百炼** 的医疗 RAG 系统，实现可溯源的专业问答；
> **第二**，基于 **VTK.js** 的浏览器端 CT 三维体渲染，无需安装插件；
> **第三**，**多模态 AI 融合**，集成了对话、搜索、视觉理解、OCR 多种模型；
> **第四**，**Docker 容器化**，一键部署，开箱即用。

> 谢谢大家！"

## 演示前准备清单

### 环境启动
- [ ] 启动后端：`npm run dev:server`（终端 1）
- [ ] 启动前端：`npm run dev:web`（终端 2）
- [ ] 确认 PostgreSQL 和 Redis 已运行
- [ ] 确认 API Key 已配置（`QWEN_API_KEY`）

### 数据准备
- [ ] 知识库已有至少 5 条文档
- [ ] 测试账号已注册（手机号 `13800138000` / 密码 `Test123456`）
- [ ] 模拟 CT 数据已缓存（首次加载后第二次更快）

### 浏览器设置
- [ ] 使用**无痕模式**打开
- [ ] 清除 localStorage（避免旧对话干扰）
- [ ] 关闭无关浏览器标签页
- [ ] 关闭系统通知

### 录屏设置
- [ ] 录制区域：浏览器窗口区域（非全屏）
- [ ] 帧率：30fps
- [ ] 分辨率：1920×1080
- [ ] 格式：MP4 (H.264)
